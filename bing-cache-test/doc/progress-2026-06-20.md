# 2026-06-20 多实例部署测试方案

## 背景

bing-cache-demo 是 bing-cache 两级缓存库（Caffeine L1 + Redis L2）的测试工程。本次工作新增了多实例部署测试能力，模拟多个服务实例共享 Redis 的场景，验证分布式缓存的一致性和可靠性。

## 已完成工作

### 1. 多实例配置文件

- `src/main/resources/application-instance1.yml` — port 8081
- `src/main/resources/application-instance2.yml` — port 8082
- `src/main/resources/application-instance3.yml` — port 8083

每个文件只覆盖 `server.port`，其他配置（Redis 连接、bing-cache 参数）继承自 `application.yml`。

### 2. 实例标识端点

在 `DemoController.java` 新增 `GET /api/instance-info`：

```json
{ "port": 8081, "hostname": "xxx", "pid": 12345, "status": "UP" }
```

用于多实例测试中区分不同实例，同时充当健康检查端点。

### 3. 一键启动脚本

`scripts/start-instances.sh`：

- 支持自定义实例数量和起始端口号
- 自动编译项目
- 等待所有实例就绪（轮询 `/api/instance-info`）
- `Ctrl+C` 优雅停止所有实例

```bash
./scripts/start-instances.sh          # 3 个实例，8081-8083
./scripts/start-instances.sh 2 9090   # 2 个实例，9090-9091
```

### 4. 多实例集成测试

`src/test/java/com/example/demo/MultiInstanceCacheTest.java`（默认 `@Disabled`）：

| # | 测试方法 | 验证要点 |
|---|----------|----------|
| 1 | `testL2CacheSharedAcrossInstances` | 实例1写缓存 → 实例2读命中 L2 Redis |
| 2 | `testEvictPropagationAcrossInstances` | 实例1 evict → 实例2收到 pub/sub 后重算 |
| 3 | `testConcurrentAccessMultipleInstances` | 3 实例并发 30 次请求，零错误 |
| 4 | `testStressMultipleInstances` | 混合查询/evict 压力测试，验证 stability |

### 5. 环境问题修复

- **问题**: Git Bash 下 `mvn` shell 脚本传 Unix 路径给 Windows Java，导致 `ClassNotFoundException`
- **根因**: Maven 的 MinGW 路径转换不完整，未调用 `cygpath`
- **解决**: `~/.bashrc` 添加 `alias mvn='.../mvn.cmd'`，强制使用 Windows 批处理版本

## 使用指南

```bash
# 终端1: 启动多实例
./scripts/start-instances.sh 3

# 终端2: 从父仓库根目录运行多实例测试（需先去掉 @Disabled 注解）
mvn -pl bing-cache-test -am -Dtest=MultiInstanceCacheTest test
```
